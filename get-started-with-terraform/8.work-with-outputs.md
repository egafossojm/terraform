# Terraform Outputs

### 1. What are output values ?
- Output values are used to EXPORT structured data about resources.
- Output values are like functions return values in programming languages.
- We can use them to share data form child modules to the root module.
- Root module can use outputs to print value at the terminal after running `terraform apply`.
- `terraform plan` will not print output values.

> [!Note]
>
> - We can declare outputs anywhere in the configuration file, But it is good practice to declare them in a separated `outputs.tf` file.

EX: `outputs.tf`
```tf
output "public_ip" {
    value = aws_instance.my_vm.public_ip
    description = "Public IP of the instance"
}

output "vpc1_id" {
    value = aws_vpc.vpc1.id
    description = "VPC1 ID"
}
```

- Now run terraform
```bash
$ terraform apply -auto-approve

...

Apply complete! Resources: 0 added, 0 changed, 0 destroyed.

Outputs:

public_ip = "3.239.196.220"
vpc1_id = "vpc-0708a64b5b5ad9508"
```

### 2. Terraform outputs

- Terraform stores output values in its **state file**
- Therefore, we can print outputs values with `terraform output` command.
```bash
$ terraform output
public_ip = "3.239.196.220"
vpc1_id = "vpc-0708a64b5b5ad9508"
```
- You may want to see only one specific output value:
```bash
$ terraform output public_ip
"3.239.196.220"
```
- You may also want to genereate json format outputs for application:
```bash
$ terraform output -json
{
  "public_ip": {
    "sensitive": false,
    "type": "string",
    "value": "3.239.196.220"
  },
  "vpc1_id": {
    "sensitive": false,
    "type": "string",
    "value": "vpc-0708a64b5b5ad9508"
  }
}
```

- You can also redirect this outputs to a json file: 
```bash
$ terraform output -json > outputs.json
$ cat outputs.json
{
  "public_ip": {
    "sensitive": false,
    "type": "string",
    "value": "3.239.196.220"
  },
  "vpc1_id": {
    "sensitive": false,
    "type": "string",
    "value": "vpc-0708a64b5b5ad9508"
  }
}
```

### 3. Sensitive Outputs
- If an output value is sensitive, you can mark it as is in the output declaration.
```bash
output "ami_id" {
    value = aws_instance.my_vm.ami
    description = "Instace AMI ID"
    sensitive = true
}
```
- Sensitive values will not appear in the `terraform output` output
```bash
$ terraform output
ami_id = <sensitive>
public_ip = "3.239.196.220"
vpc1_id = "vpc-0708a64b5b5ad9508"
```
>[!WARNING]
>
> - Sensitive output value will appear when you print specific output.
>  ```bash
> $ terraform output ami_id
> "ami-03a4de20979bca479"
> ```
>
> - Will also appear in json
> ```bash
