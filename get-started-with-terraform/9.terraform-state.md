# Terraform State

### 1. How it works ?

- Terraform stores state information about your mmanaged infrastructure.
- This state is used by Terraform to `map real-world resources` to your configuration, to `keep track of metatdata` and to `improve performance`.
- The state is stored ny default in a file called `terraform.tfstate`. But it can also be stored remotly, which works better in a **team environment**.
- This file does not exist until you apply at least one `terraform apply`.

>[!WARNING]
>
> - Even if `terraform.tfstate` is editable, it is not recommended to directly edit it.
> - Terraform provides some commands if you need to make basic modifications on it.

- Terraform compares, **state file**, **configuration files** and **real infrastructure** to make changes.
- A backup of `terraform.tfstate` file is stored as `terraform.tfstate.backup` in case you lost the terraform.tfstate file.
- When terraform creates an object, Its requests its ID in the state.
- When terraform destroy an object, its removes the binding for it in the state.
- When you run ` terraform apply` or `terraform destroy` agains your configuration, terraform writes metadata about your configuration in the state file, and update you infrastructure resources accordingly.

### 2. terraform.tfstate content

In the `terraform.tfstate` file, you find:

-  The terraform version in use
- `resources` section that contains all the resources managed by terraform.
* `outputs` section


`terraform.tfstate`
```
{
  "version": 4,
  "terraform_version": "1.14.9",
  "serial": 120,
  "lineage": "cf5306b3-301b-272a-b26f-4e875eb241ef",
  "outputs": {},
  "resources": [],
  "check_results": null
}

```


Each resource has the following key/values:
* `mode`: **data** for data sources or **managed** for managed resources.
* `type`: the type of the data source or the managed resource.
* `name`: The given local name of the resource/data source. 
* `provider`: resource/data source provider
* `instances`: the section that contains the attributes of the resource including its **dependencies**.

### 3. terraform state commands

These commnds allow to revieuw the resources in the state without interacting directly with the state file.

- `terraform show` : output in json, ALL the resources in the state.


>[!NOTE]
>
> - Note that terraform state can contain sensitive data like **database passwords**, **user passwords** or **private keys** since its stores the resource IDs and those resource attributes in palin text json files.
> - Storing state remotely can provide better security.
> - When using  remote state, state is only held in memory when used by terraform.
