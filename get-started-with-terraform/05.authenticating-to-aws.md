# Authenticating to AWS

Oredered by precendent (Where terraform will go and find credentials first)

### 1. Add credentials in provider block

>[!WARNING]
>
> - Never do this for production
> - Never add this to the version control systems

```tf
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 6.0"
    }
  }
}

# Configure the AWS Provider
provider "aws" {
  region     = "ca-central-1"
  access_key = "AKIA4FRSSAAA2FGRCCCC"
  secret_key = "icNrkisdTomxxxxxxTaO/vZWRyGfQUYYYb7Mu7zJ"
}
```

### 2. Add credentials in environment variables
- If `provider` block is empty, Terraform will look for env variables in the your terminal.
- Set them temporarily
```bash
export AWS_ACCESS_KEY_ID="AKIA4FRSSAAA2FGRCCCC"
export AWS_SECRET_ACCESS_KEY="icNrkisdTomxxxxxxTaO/vZWRyGfQUYYYb7Mu7zJ"
export AWS_DEFAULT_REGION="ca-central-1"
```

- Set them permanently
```bash
cat <<EOF >>  ~/.bashrc
export AWS_ACCESS_KEY_ID="AKIA4FRSSAAA2FGRCCCC"
export AWS_SECRET_ACCESS_KEY="icNrkisdTomxxxxxxTaO/vZWRyGfQUYYYb7Mu7zJ"
export AWS_DEFAULT_REGION="ca-central-1"
EOF
```

### 3. Add credentials in ~/.aws/credentials
- If terraform can't find credentials in `provider` bloc and cli env variables, it will rely on **AWS CLI** config
- It will read the `default` profile or the one specified by `$AWS_PROFILE`
- To configure AWS CLI, run `aws configure`
- This stores credentials in `~/.aws/credentials` on your local machine.
- Terraform uses `~/.aws/credentials` and `~/.aws/config`

### 4. Use IAM role if working on EC2 Instance.
- Terraform uses metadata of the instance profile (EC2/ECS)
- Best security practice
