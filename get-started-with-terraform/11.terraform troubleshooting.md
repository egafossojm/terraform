# Terraform troubleshooting

### 1. Terraform format
Get a best view of configuration file, run `terraform fmt` to make configurattion readable.

### 2. Terraform validate
`terraform validate` makes sure there is not any error in the configuration files.

### 3. Terraform Logging
If there are provider errors or any other error that still persists, you can enable ** Detailed logging**. You can also store logs in a specified file.

#### - Enable terraform logging
EX:
```bash
export TF_LOG=DEBUG
```

#### - Logging levels
- `TRACE`: Most verbose
- `DEBUG`
- `INFO`
- `WARN`
- `ERROR`: Least verbose, Only errors
- `OFF`: Default

#### - Set terraform log file
You can keep logs for long time buy setting up a LOG FILE.

```bash
export TF_LOG_PATH=/home/user/terraform.log
```

`terraform refresh`, `terraform plan` and `terraform apply` will append thier output into the specified log file.

#### - Core logs level vs Provider logs level
You can decide to log terraform core logs and terraform provider logs at `different levels`:
```bash
export TF_LOG_CORE=INFO

export TF_LOG_PROVIDER=DEBUG
```
