# Terraform Providers

### 1. What is a  Terraform Provider ?
- This is is a `plugin` that enables Terraform to interact with cloud providers
 (AWS, Azure, GCP), SaaS providers, and other APIs to manage resources.
 - Ther are providers fir IaaS, PaaS and SaaS.
 - They are separated from terraform it self and each provider has its own release date and version number.
 - Publicly available providers can be found at [Terraform Providers](https://registry.terraform.io/search/providers). for most major infrastructure plateforms.

 ### 2. Provider tiers
 - **Official providers**: Owned, developped and publish by Hashicorp
 - **Verified providers**: Published by 3rd parties, Technology parners and verified by Hashicorp.
 - **Community providers**: Published by users or volonteers

 ### 3. Configure a Provider (AWS provider in this case)
 - For each provider, there is documentation that shows What is the `provider class`, What `resources` it provides and how to `use it`.

 >[!NOTE]
 >
 > Provider configurations belongs to the **root module** of the terraform project

```tf
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 6.0"
    }
  }
}

# Configure the AWS Provider
provider "aws" {
  region = "us-east-1"
}
```
- In this config, `aws` is a **local name** given to the provider. It must match the provider name in provider configuration section.
- In this config, `source` the **global name** for the provider.
