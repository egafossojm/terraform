# Terraform Data Sources

- Each provider may offer data sources along side of set of resource types.
- In the terraform documentation for a given provider (Let's say AWS), You will see for each service `Resource` and `Data Sources`.

### 1. What are Data Sources ?
- They are kind of API and their goal is to do something and give you back Dynamic datasources.
- They are API that fetches dynamic data from cloud providers.
- Data sources are like queries.\
- > EX: 
>- List of AMI that changes frequently
>- List of Availability Zones

### 2. Example

```tf
data "aws_ami" "amazon_linux_2023" {
  most_recent = true
  owners      = ["amazon"]

  filter {
    name   = "name"
    values = ["al2023-ami-2023*-x86_64"]
  }

  filter {
    name   = "architecture"
    values = ["x86_64"]
  }
}

resource "aws_instance" "my_vm" {
  ami = data.aws_ami.amazon_linux_2023.image_id
  instance_type = "t2.micro"
  vpc_security_group_ids = [aws_default_security_group.default_sg.id]
  associate_public_ip_address = true
  key_name = aws_key_pair.test_ssh_key.key_name 
  subnet_id = aws_subnet.web.id
  tags = {
    "Name" = "${var.main_vpc_name} VM"
  }
```

- This will work in any aws region.
- The right Amazon Linux 2023 AMI ID will be automatically found in any region.
