# Terraform types
There are 2 categories of variable in terraform: Simple and Complex.
Setting types makes terraform errors pricise.

## Simple types
### 1. number
- numerical values
- They do no need quotes.
```tf
variable "web_port" {
  description = "WEB PORT"
  type = number
  default = 80
}

resource "aws_default_security_group" "default_sg" {
  vpc_id = aws_vpc.vpc1.id
  ingress {
    from_port   = var.web_port
    to_port     = var.web_port
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
    #cidr_blocks = [var.my_public_ip]
  }

  tags = {
    "Name" = "${var.main_vpc_name} Default SG"
  }
}
```

### 2. string
- requires double-quotes

```tf
variable "aws_region" {
  description = "AWS regions"
  type = string
  default = "ca-central-1"
}

terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 6.0"
    }
  }
}

# Configure the AWS Provider
provider "aws" {
  region = var.aws_region
}
```

### 3. bool
- Its values can be `true` or `false`
- does not requite quotes.

```tf
variable "enable_dns" {
  description = "Enable dns support"
  type = bool
  default = true
}

resource "aws_vpc" "vpc1" {
  cidr_block = var.vpc_cidr_block
  enable_dns_support = var.enable_dns
  tags = {
    Name = "Production ${var.main_vpc_name}"
  }
}
```

## Complex types
### 1. Collection types
Groups values of the `same type`.

#### a. type list
- A list is a sequence of values of the same type.
- Identify elements in a list with consecutive whole numbers, starting with zero.
```tf
variable "azs" {
  description = "Availability zones"
  type        = list(string)
  default = [
    "ca-central-1a",
    "ca-central-1b",
    "ca-central-1c"
  ]
}

resource "aws_subnet" "web" {
  vpc_id            = aws_vpc.vpc1.id
  cidr_block        = var.web_subnet
  availability_zone = var.azs[0]

  tags = {
    Name = "web-subnet"
  }
}
```

#### b. type map
- Map is a collection of key:value pairs. ALL of the same type
```tf
variable "amis" {
  type = map(string)
  default = {
    "ca-central-1" = "ami-0495a76ecf381a767",
    "us-east-1" = "ami-098e39bafa7e7303d"
  }
}
```
- You can also avoid to specify the map type (BUT NOT RECOMMENDED !!!)
```tf
variable "amis" {
  type = map
  default = {
    "ca-central-1" = "ami-0495a76ecf381a767",
    "us-east-1" = "ami-098e39bafa7e7303d"
  }
}
```
- Use the map
```tf
resource "aws_instance" "my_vm" {
  #ami                         = data.aws_ami.amazon_linux_2023.image_id
  ami                         = var.amis[var.aws_region]
  instance_type               = "t2.micro"
  vpc_security_group_ids      = [aws_default_security_group.default_sg.id]
  associate_public_ip_address = true
  key_name                    = aws_key_pair.test_ssh_key.key_name
  subnet_id                   = aws_subnet.web.id
  user_data = file("entry-script.sh")

  tags = {
    "Name" = "${var.main_vpc_name} VM"
  }
}
```

#### c. type set
- A collection of unique values that do not have any secondary identifiers or ordering.
- Terraform does not support directly accessing elements of a set by index because `Sets are unordered collections`.
- Define a set
```tf
variable "example_set" {
  type    = set(string)
  default = ["foo", "bar"]
}
```
- Use the `tolist` function to convert the set to a list.
```tf
locals {
  example_list = tolist(var.example_set)
}
```
- You can then reference an element in the list
```tf
output "first_element" {
  value = local.example_list[0]
}
output "second_element" {
  value = local.example_list[1]
}
```
